# Migration Templates

This directory contains templates used by the clean-break migration policy.

## Templates

- `template.clean-break-migration.md`
- `release-notes.template.md`

## Canonical Destination

Clean-break migration plans generated from this template must be stored in:

- `/.harmony/cognition/runtime/migrations/<YYYY-MM-DD>-<slug>/plan.md`

Migration evidence generated by those plans must be stored as bundles in:

- `/.harmony/output/reports/migrations/<YYYY-MM-DD>-<slug>/`
- required files:
  - `bundle.yml`
  - `evidence.md`
  - `commands.md`
  - `validation.md`
  - `inventory.md`
