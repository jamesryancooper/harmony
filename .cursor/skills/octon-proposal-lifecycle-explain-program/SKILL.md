# Octon Proposal Lifecycle: Explain Program

Explain a parent proposal program's scope, child registry, sequencing,
dependencies, gates, aggregate evidence posture, closeout posture, and child
authority boundaries without changing lifecycle state or promoting
proposal-local analysis as authority.

Always disclose that child manifests, subtype manifests, receipts, validation
verdicts, promotion targets, acceptance criteria, and archive metadata remain
child-owned. Parent evidence may summarize child outcomes but never satisfies
child receipts or child authority.
