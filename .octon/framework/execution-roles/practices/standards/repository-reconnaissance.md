# Repository Reconnaissance

## Purpose

Require search-before-create discipline for AI-assisted implementation so new
surfaces are added only when existing Octon surfaces are the wrong home.

## Required Search Before New Surfaces

Before creating a helper, type, validator, contract, policy, workflow,
practice, generated output, dependency, or proposal, search the relevant local
families:

- constitutional and authority contracts;
- instance governance policies and contracts;
- ingress and bootstrap surfaces;
- execution-role practices and standards;
- runtime specs and adapters;
- capability packs, skills, commands, and services;
- assurance validators, suites, fixtures, and workflows;
- generated projections and publication metadata;
- proposal packets for lineage only;
- Rust workspace crates and shell scripts;
- GitHub projection workflows when the change is repo-local projection work.

## Receipt

Record:

- searches run;
- existing surfaces found;
- reused surfaces;
- rejected surfaces and reason;
- new surfaces proposed and why no existing home is correct.

## Fail-Closed Cases

Block or escalate when:

- a new surface duplicates an existing authority, validator, workflow, or
  helper;
- a proposal packet is treated as durable authority;
- generated discovery is treated as policy or runtime truth;
- ownership of the proposed durable home is ambiguous.
